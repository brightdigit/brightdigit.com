<p><a
href="https://brightdigit.com/episodes/203-milk-diary-with-kaya-thomas/"
target="_blank">Great to have <strong>Kaya Thomas</strong> back on
EmpowerApps!</a><span style="color:rgb(0, 0, 0);"> You might remember
her from </span><a
href="https://brightdigit.com/episodes/041-notifications-with-kaya-thomas/"><span
style="color:rgb(0, 0, 0);">our 2020 episode about
notifications</span></a><span style="color:rgb(0, 0, 0);">- she's now
back as a founder building MilkDiary after becoming a mom last
year.</span></p>
<p><span style="color:rgb(0, 0, 0);">MilkDiary tackles comprehensive
feeding tracking for the first year of parenting. Kaya found existing
apps too rigid and fragmented - they might handle one aspect well but
fail when feeding methods change or you need to track the full journey
from pumping to storage to feeding.</span></p>
<p><strong><span style="color:rgb(0, 0, 0);">Key technical
highlights:</span></strong></p>
<ul>
<li><p><strong><span style="color:rgb(0, 0, 0);">Foundation
Models</span></strong><span style="color:rgb(0, 0, 0);">: Uses iOS 26's
on-device models for voice/text logging with guided generation, avoiding
cloud privacy concerns</span></p></li>
<li><p><strong><span
style="color:rgb(0, 0, 0);">AlarmKit</span></strong><span
style="color:rgb(0, 0, 0);">: Replaces notifications for immediate
actions that shouldn't be easily dismissed</span></p></li>
<li><p><strong><span style="color:rgb(0, 0, 0);">Speech
Analyzer</span></strong><span style="color:rgb(0, 0, 0);">: Leverages
the improved on-device transcription model</span></p></li>
<li><p><strong><span style="color:rgb(0, 0, 0);">Data
Architecture</span></strong><span style="color:rgb(0, 0, 0);">: Moved
from SwiftData to GRDB due to CloudKit sharing limitations and predicate
handling issues</span></p></li>
<li><p><strong><span style="color:rgb(0, 0, 0);">Multiple
Children</span></strong><span style="color:rgb(0, 0, 0);">: Supports
twins/multiples without profile switching</span></p></li>
</ul>
<p><span style="color:rgb(0, 0, 0);">Kaya's approach of "building in
community" - involving target users throughout development - really
resonated with me. She's also using Claude for code review and test
generation, treating it like a helpful intern rather than a replacement
for thinking. </span><a
href="https://brightdigit.com/episodes/203-milk-diary-with-kaya-thomas/"
target="_blank">Catch this episode here:</a></p>

<p><a
href="https://brightdigit.com/episodes/202-swift-testing-with-rachel-brindle/"
target="_blank"><strong>Rachel Brindle</strong> joined us to discuss
Swift Testing's maturity and what's new in Swift 6.2.</a> <span
style="color:rgb(0, 0, 0);">As maintainer of Quick, Nimble, and Swift
Fakes, plus her role on the Swift Testing Workgroup, Rachel has deep
insight into the testing ecosystem.</span></p>
<p><strong><span style="color:rgb(0, 0, 0);">Swift Testing
advantages:</span></strong></p>
<ul>
<li><p><span style="color:rgb(0, 0, 0);">Better DSL with </span><span
style="color:rgb(0, 0, 0);"><code>@Test</code></span><span
style="color:rgb(0, 0, 0);"> macro and human-readable function names
(using backticks for spaces/punctuation)</span></p></li>
<li><p><span style="color:rgb(0, 0, 0);">Concurrent execution by default
(forces better test isolation)</span></p></li>
<li><p><span style="color:rgb(0, 0, 0);">Open development process
through the Testing Workgroup</span></p></li>
</ul>
<p><strong><span style="color:rgb(0, 0, 0);">Migration
considerations:</span></strong><span style="color:rgb(0, 0, 0);"> Tests
using shared global state are difficult to migrate due to concurrent
execution. Rachel recommends starting new components with Swift Testing
while keeping existing XCTest suites in place.</span></p>
<p><strong><span style="color:rgb(0, 0, 0);">Swift 6.2 new
features:</span></strong></p>
<ul>
<li><p><strong><span style="color:rgb(0, 0, 0);">Exit
Tests</span></strong><span style="color:rgb(0, 0, 0);">: Verify
command-line tools exit correctly (non-iOS only due to
sandboxing)</span></p></li>
<li><p><strong><span style="color:rgb(0, 0, 0);">Range
Confirmations</span></strong><span style="color:rgb(0, 0, 0);">: Expect
callbacks within ranges (useful for automation tests)</span></p></li>
<li><p><strong><span
style="color:rgb(0, 0, 0);">Attachments</span></strong><span
style="color:rgb(0, 0, 0);">: Add debugging artifacts to test
results</span></p></li>
</ul>
<p><strong><span style="color:rgb(0, 0, 0);">SwiftUI Testing
Reality</span></strong><span style="color:rgb(0, 0, 0);">: Still limited
to automation tests. While tools like ViewInspector help, first-party
unit testing support remains absent despite frequent WWDC lab requests.
Rachel's working on polling confirmations (think </span><span
style="color:rgb(0, 0, 0);"><code>expect(...).eventually.to(equal(...))</code></span><span
style="color:rgb(0, 0, 0);">) which could be game-changing for async
testing patterns. Check out the episode here:<br />
</span><a
href="https://brightdigit.com/episodes/202-swift-testing-with-rachel-brindle/"
target="_blank">https://brightdigit.com/episodes/202-swift-testing-with-rachel-brindle/</a></p>
